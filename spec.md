# エンジニア教育管理システム 仕様書（検討中）

> このドキュメントは壁打ち相談中の仕様メモです。随時更新します。

---

## 1. システム概要

新卒エンジニアの教育・育成を管理するWebシステム。
チェックリストによる習得確認・S/A/B/C評価・課題提出と採点・進捗ダッシュボードを中心とする。
将来的に外部SaaS展開も視野に入れた設計とする。

---

## 2. ユーザーロール

| ロール | 説明 | 主な操作 |
|--------|------|---------|
| 受講者 | 対象エンジニア（新卒など） | チェックリスト確認、課題提出、進捗確認 |
| 講師 / メンター | 評価・課題レビュー担当 | 評価入力、課題採点、カリキュラム管理 |
| 管理者 | システム全体管理 | ユーザー管理、チェックリスト編集、全体把握 |
| HR担当者 | 採用・育成管理担当 | レポート閲覧、進捗ダッシュボード閲覧 |

---

## 3. グループ・組織構造

将来のマルチテナント（SaaS）を見据えた階層設計。

```
テナント（会社）               ← SaaS展開時に複数社対応
 └─ 年度コーホート              例：2024年度入社、2025年度入社
     └─ 部署・チーム            例：開発部、インフラチーム
         └─ 受講者              担当講師が割り当てられる
```

## 5. 学習コンテンツ（ハイブリッド型）

各チェックリスト項目に「学習ガイド」を紐付ける。詳細コンテンツは外部リンクに委ねる。

```
チェックリスト項目
 └─ 学習ガイド（管理者・講師がMarkdownで編集可）
     ├─ 概要・学習のポイント（テキスト）
     ├─ 参考リンク（複数URL、タイトル付き）
     └─ 対応する課題・テスト（紐付け）
```

### 運用イメージ

| 項目 | 内容 |
|------|------|
| 学習ガイド | 「何を・なぜ学ぶか」の概要をMarkdownで記述 |
| 参考リンク | Zenn記事・社内Notion・公式ドキュメント等のURL |
| 動画等 | 外部URL（YouTube・Udemy等）のリンクとして管理 |
| 閲覧ログ | **管理しない**（シンプルさを優先。将来拡張可） |

## 6. 課題・テスト管理

### 6.1 課題タイプ（デフォルト）

| テスト | 内容 |
|--------|------|
| Git試験 | branch作成 + PR提出 |
| SQL試験 | 3テーブルJOIN |
| プログラム試験 | 簡易API作成 |
| デバッグ試験 | バグ修正 |

### 6.2 提出方法

- GitHubリポジトリURL
- テキスト記述回答
- （将来）ファイルアップロード

### 6.3 課題ステータスフロー

```
未着手 → 提出済 → レビュー中 → 評価完了
                               └─ 差し戻し → 提出済（再提出）
```

### 6.4 採点

- 講師が手動でフィードバック・スコア入力
- （将来）Claude API による自動採点・コードレビュー補助

## 7. 自己評価機能

### 7.1 方式：評価タイミング連動型

評価期間（1ヶ月 / 3ヶ月 / 6ヶ月 / 12ヶ月）が開始されたら、受講者が先に自己評価を入力し、その後に講師が評価する。

```
評価タイミング開始
 ↓
① 受講者：各チェックリスト項目に自己評価（S/A/B/C）＋コメントを入力
 ↓
② 講師：受講者の自己評価を参照しながら講師評価（S/A/B/C）＋コメントを入力
 ↓
③ 双方がギャップ（差分）を確認 → 面談・フィードバックへ
```

### 7.2 ギャップ表示

| 項目 | 自己評価 | 講師評価 | 差分 |
|------|---------|---------|------|
| git branch | A | B | ↓ 過大評価 |
| SQL JOIN | C | B | ↑ 過小評価 |
| Docker基本 | B | B | ✓ 一致 |

- 過大評価・過小評価の件数をサマリで表示
- 面談の話題出しとして活用できる

### 7.3 制約

- 評価タイミング期間外は自己評価を入力できない（読み取りのみ）
- 講師が評価を完了するまで、受講者は講師評価を閲覧できない（先入観防止）

## 8. 通知機能

### 8.1 方式

システム内通知（ヘッダーのベルアイコン等）。メール・Slack等の外部連携はなし。
未読件数をバッジで表示し、通知一覧画面で確認・既読管理する。

### 8.2 通知トリガー一覧

| 受信者 | トリガー | 通知内容 |
|--------|---------|---------|
| 受講者 | 評価タイミング開始 | 「〇ヶ月評価が始まりました。自己評価を入力してください」 |
| 受講者 | 講師評価が完了した | 「講師が評価を入力しました。確認してください」 |
| 受講者 | 課題が差し戻された | 「課題『Git試験』が差し戻されました。フィードバックを確認してください」 |
| 講師 | 受講者が自己評価を完了した | 「〇〇さんが自己評価を入力しました。評価を入力してください」 |
| 講師 | 課題が提出された | 「〇〇さんが課題『SQL試験』を提出しました」 |
| 管理者 | 未評価が〇日以上放置 | 「〇〇さんの評価が〇日間未入力です」 |

### 8.3 データモデル

```
Notification
 ├─ user_id（受信者）
 ├─ type（通知種別）
 ├─ message（通知文）
 ├─ link（関連ページへのURL）
 ├─ is_read（既読フラグ）
 └─ created_at
```

## 9. 進捗ダッシュボード

### 受講者向け
- カテゴリ別 S/A/B/C 分布（レーダーチャートなど）
- 課題提出状況・評価履歴
- 評価タイミングまでの残日数

### 講師向け
- 担当受講者一覧と進捗サマリ
- 未評価チェックリスト・未採点課題の通知
- 受講者比較

### 管理者 / HR向け
- 年度コーホート別・部署別の進捗集計
- 課題合格率・評価分布
- レポートエクスポート（CSV / PDF）

## 11. データモデル（概要）

```
User（ユーザー）
 ├─ role: learner / instructor / admin / hr
 ├─ CohortYear（年度）
 └─ Department（部署）

ChecklistTemplate（チェックリストテンプレート）
 └─ ChecklistItem（項目）× 100
     └─ LearningGuide（学習ガイド）
         ├─ body（Markdown本文）
         └─ ResourceLink（参考リンク）× 複数
             ├─ title
             └─ url

LearnerChecklist（受講者別チェックリスト）
 └─ LearnerChecklistItem
     ├─ self_rating: S / A / B / C / 未評価（受講者入力）
     ├─ self_comment（受講者コメント）
     ├─ instructor_rating: S / A / B / C / 未評価（講師入力）
     ├─ instructor_comment（講師コメント）
     └─ evaluation_period_id（どの評価タイミングか）

Notification（通知）
 ├─ user_id（受信者）
 ├─ type（通知種別）
 ├─ message
 ├─ link
 ├─ is_read
 └─ created_at

Assignment（課題）
 └─ Submission（提出物）
     ├─ github_url / text_answer
     └─ Review（採点結果）
         ├─ score
         └─ feedback

EvaluationPeriod（評価タイミング）
 ├─ 1ヶ月 / 3ヶ月 / 6ヶ月 / 12ヶ月
 └─ LearnerEvaluation（評価サマリ）
```
