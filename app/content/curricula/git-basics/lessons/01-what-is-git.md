## Gitとは

Gitは**バージョン管理システム**です。ファイルの変更履歴を記録し、過去の状態に戻したり、複数人で共同作業をしたりすることができます。

### なぜGitを使うのか

- **変更履歴の管理**：いつ・誰が・何を変えたかを記録できる
- **バックアップ**：リモートリポジトリに保存することで、PCが壊れてもデータを守れる
- **チーム開発**：複数人が同じコードを並行して修正し、最終的に統合できる

### 基本的な概念

| 用語 | 説明 |
|------|------|
| リポジトリ（repo） | 変更履歴を保存する場所 |
| コミット | 変更のスナップショット |
| ブランチ | 作業の分岐 |
| マージ | ブランチの統合 |

### ローカルとリモート

```
ローカルリポジトリ（自分のPC）
        ↕ push / pull
リモートリポジトリ（GitHub等）
```

> ポイント：まずは「変更を記録するもの」として理解しましょう
